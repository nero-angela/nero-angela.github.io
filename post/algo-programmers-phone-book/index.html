프로그래머스의 알고리즘 문제 “전화번호 목록”를 Python3를 이용하여 풀어보도록 하겠습니다.

# Problem

출처 : Programmers

문제 설명

전화번호부에 적힌 전화번호 중, 한 번호가 다른 번호의 접두어인 경우가 있는지 확인하려 합니다.
전화번호가 다음과 같을 경우, 구조대 전화번호는 영석이의 전화번호의 접두사입니다.

전화번호부에 적힌 전화번호를 담은 배열 phone_book 이 solution 함수의 매개변수로 주어질 때, 어떤 번호가 다른 번호의 접두어인 경우가 있으면 false를 그렇지 않으면 true를 return 하도록 solution 함수를 작성해주세요.

제한 사항

  • phone_book의 길이는 1 이상 1,000,000 이하입니다.
  • 각 전화번호의 길이는 1 이상 20 이하입니다.

입출력 예제

phone_book return
[119, 97674223, 1195524421] false
[123,456,789] true
[12,123,1235,567,88] false

입출력 예 설명

입출력 예 #1
앞에서 설명한 예와 같습니다.

입출력 예 #2
한 번호가 다른 번호의 접두사인 경우가 없으므로, 답은 true입니다.

입출력 예 #3
첫 번째 전화번호, “12”가 두 번째 전화번호 “123”의 접두사입니다. 따라서 답은 false입니다.

# Solution

Iterator : O(n^2)

시작 복잡도가 O(n^2)이라 효율성 테스트에서 시간초과할 줄 알았는데 통과했네요. 🤔

Hash : O(n^2)

0 ~ 9까지의 값을 key로 갖는 dict를 만들고, 내부에서만 비교하도록 구현해 보았습니다.

조금 더 나아지긴 했지만, 첫 번째 알고리즘을 이길 순 없는 것 같습니다.

Hash : O(n^2)

0 ~ 9까지의 값을 key로 갖는 dict를 만들고, 해당 key로 시작하는 숫자들을 오름차순으로 넣습니다. 그리고 각 value 안에서 자신보다 더 큰 값이 startswith() 하는지만 비교하도록 구현해 보았습니다.

두 번째 알고리즘 보다 안좋아졌습니다. 😓

String Sort & Zip : O(n^2)

string 형태로 된 숫자를 정렬 할 경우, 크기와 관계없이 문자로 취급하기 때문에 동일하게 시작되는 숫자는 인접하게 정렬되는 것을 이용한 알고리즘입니다.

데이터의 상태에 따라 다르겠지만, 여기서는 처음 생각한 방법이 제일 좋은 방법인 것 같습니다.
